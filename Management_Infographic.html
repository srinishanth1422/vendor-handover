Transformation

Before vs. After Automation

❌ Previous Workflow
• Genpact raised individual tickets for every repush
• SnapLogic processed each ticket manually (15–20 min)
• Response often delayed to next business day
• Single PRID per request only
• Full dependency on EUIT SnapLogic team availability
• Manual timestamp retrieval and sharing
→
✅ Current Automated Workflow
• Self-service trigger via web application
• SnapLogic pipeline executes automatically (~5 min)
• Near-real-time processing, no waiting
• Multi-PRID batch processing supported
• Dependency-free, operator-driven execution
• Auto-retrieval of vendor push timestamps
✅

Automation ROI

Per-ticket handling reduced from 15–20 minutes (+ next-day lag) to ~5 minutes of automated processing.
EUIT SnapLogic team freed from 7–8 daily manual repush tasks, enabling focus on higher-value integration work.

SnapLogic Automation Impact Summary

By automating the vendor repush process, SnapLogic eliminated 543 hours/year of manual ticket processing and removed an average 24-hour response lag from every request.

The team now processes the same 1,630 annual tickets in ~136 hours of automated pipeline time (1,630 × 5 min), freeing 407 hours annually for higher-value integration development and pipeline optimization.

Procedures

Standard Operating Procedures & Escalation

Vendor Creation Workflow
1
Ticket CreationGenpact creates vendor creation ticket and routes to SnapLogic
2
Data Retrieval & ValidationSnapLogic fetches bank details from Snowflake HR RAW; validates Acct No (17 chars), IBAN, IFSC, mandatory fields
3
User Follow-Up (if needed)If validation fails → contact user to update Workday → wait for HR RAW refresh (up to 1 hr) → re-verify
4
Vendor PushAfter validation, SnapLogic creates the vendor and generates a timestamp, which Genpact shares with SAP PI.
5
SAP PI VerificationSAP PI validates vendor data ingestion and confirms receipt
6
SAP FI Review & ClosureSAP FI performs final financial review → Genpact closes ticket
Escalation Matrix
User in STOP Table
Remove from STOP table and repush vendor creation
→ Escalate to: SAP FI Team
SAP Bank Validation Failure
Contact user to correct bank details in Workday; re-verify after HR RAW refresh
→ Escalate to: Genpact → End User
User on Leave / Termination
Verify employment status; coordinate with HR for resolution
→ Escalate to: HR RAW Team
HR RAW Data Discrepancy
SnapLogic cannot view bank details; escalate with SAP failure comments
→ Escalate to: HREUITDataSupport@astrazeneca.com
Vendor Push Timestamp Unavailable
Wait 5 minutes and retry; if persistent, contact SnapLogic support
→ Escalate to: azeusnaplogicsupport@astrazeneca.com
